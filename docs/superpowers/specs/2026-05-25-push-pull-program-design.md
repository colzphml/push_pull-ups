# Программа отжиманий и подтягиваний с нуля — дизайн

**Дата:** 2026-05-25
**Автор:** Claude + colz
**Статус:** на вычитку

---

## 1. Контекст и вводные

Персональная программа силовой работы с собственным весом, построенная как «сестра» проекта `health_food`: GitHub Pages, PWA, недели карточками, скилл-генератор и SQLite-история. Тема — **отжимания (push) и подтягивания (pull)**, а не еда.

**Исходные данные пользователя:**
- Вес ~130 кг, снижается (см. `health_food`). Каждый сброшенный кг облегчает push/pull — проекты синергичны.
- Отжимания: полноценные тяжело → старт с **P0 (от стены)**, быстрый переход дальше при лёгкости.
- Подтягивания: 0 повторов, вис ~20 сек.
- **Турник дома есть**, но висит высоко (фиксированный) → классические горизонтальные тяги под низкой перекладиной невозможны.
- Резинок нет — будут куплены (см. §9).
- Днём по-разному (дом/офис) → привязка к гибким окнам, не к часам.
- Готов заниматься **каждый день**.

**Главный принцип режима — Greasing the Groove (GtG):** частые короткие подходы **вне отказа** (~40-60% от максимума), разнесённые по дню. Цель сейчас — **система/привычка** и фундамент, не рекорды.

## 2. Цели и не-цели

**Цели:**
- Дать конкретную программу на каждый день: 4-5 мини-сессий, push и pull раздельно по окнам дня.
- Вести прогресс и обратную связь между неделями; корректировать следующую неделю на их основе.
- В фоне держать общий план прогрессии (лестницы уровней + критерии перехода).
- Тот же UX, что и `health_food`, но с отдельной (холодной) цветовой темой.

**Не-цели:**
- Не зал, не штанги, не кардио-план, не питание (питание — в `health_food`).
- Не работа до отказа, не «убойные» тренировки.
- Не жёсткое расписание по часам — только окна Утро/День/Вечер.

## 3. Методика (мастер-план в `plan/progression.md`)

### Правила GtG
- Рабочий подход = ~40-60% от текущего максимума, всегда с запасом 3-4 повтора «в баке».
- Техника > количество. Каждое повторение чистое и подконтрольное.
- Подходы разнесены по дню, между однотипными блоками — время на восстановление.
- Прогресс линейный: добавляем повтор/секунды, когда текущее даётся легко. Уровень упражнения меняем по milestone.
- Боль (не путать с лёгким дискомфортом мышц) = стоп и замена/регрессия. Заносится правилом `-2`.

### Лестница PUSH (отжимания)
| Уровень | Упражнение |
|---|---|
| **P0 — старт** | От стены |
| P1 | От высокой опоры (подоконник/стол) |
| P2 | От низкой опоры (диван/устойчивый стул) |
| P3 | С колен |
| P4 | Негативы в полный рост (опускание 3-5 сек) + с колен |
| P5 | Полные отжимания |
| P6+ | Объём полных → узкие, с паузой, archer (позже) |

### Лестница PULL (подтягивания)
| Уровень | Упражнение |
|---|---|
| **H0 — старт** | Вис пассивный (dead hang), ~20 сек |
| H1 | Вис активный (лопатки вниз-назад) |
| H2 | Подтягивание лопатками (scapular pulls) |
| H3 | Горизонтальные тяги — **резинкой** (band rows) или под крепким столом; позже кольца на высокий турник |
| H4 | Негативы с резинкой (опускание сверху 3-5 сек) |
| H5 | Подтягивания с резинкой (band-assisted) |
| H6 | Полные подтягивания |
| H7+ | Объём |

> **Адаптация под высокий турник:** низкой перекладины нет, поэтому H3 делаем резинкой (band rows) — лента закреплена низко, тянем к себе; альтернатива — тяга под крепким столом (проверить надёжность при 130 кг). Кольца/петли на высокий турник — опциональная покупка позже.

> **Адаптация под 130 кг:** к полным подтягиваниям идём через объём виса, негативы и band-assisted. Прыжки к верхней точке высокого турника на старте не используем (жёстко для суставов) — помогает резинка.

### Milestones перехода (примерные, скилл уточняет по факту)
- PUSH: 3 подхода × 8-10 чисто и легко на свежесть → следующий уровень.
- HANG: вис 3 × 30 сек уверенно → активный вис + scapular + начинаем негативы.
- ROWS: 3 × 8-10 горизонтальных → негативы / band-assisted подтягивания.
- BAND PULL-UP: уверенно с лентой → более тонкая лента → первое полное.

### Суставы и восстановление
- Запястья (под отжимания) и плечи (под вис) мобилизуем **ежедневно**, особенно в лёгкие дни.
- 1-2 лёгких/восстановительных дня в неделю: короткий вис + мобилити, объём отжиманий вдвое меньше.

## 4. Структура дня = 4-5 карточек по окнам

Окна гибкие: **Утро · День · Вечер** (+ опц. «ещё подход», когда дома). push и pull чередуются.

Пример обычного дня (стартовые цифры калибруются в неделе 1):
- 🟦 **Утро · Подтягивания** — вис 3 × 20-25 сек
- 🟧 **День · Отжимания** — от стены 3 × 6-8 (запас 3-4)
- 🟦 **День/Вечер · Подтягивания** — активный вис / band rows 3 × 8
- 🟧 **Вечер · Отжимания** — от стены 3 × 6-8
- 🟪 **Перед сном · Мобилити** — запястья + плечи 3-4 мин

**Карточка** (как в `health_food`):
- **Лицо:** цветная точка (push/pull/mobility) + тип блока + суть («3×6 от стены») + подсказка по усилию.
- **Оборот (модалка):** точные подходы/повторы/секунды; 2-3 пункта техники (cues); «тяжело → регрессия»; «легко → прогрессия».

**Цвета точек:** pull = синий, push = янтарный, mobility = фиолетовый, лёгкий/отдых = серо-зелёный.
**Подсветка** «сегодня» и приглушение прошедших дней — через `data-date` (как в `health_food`).

## 5. Визуальная тема (отдельная от health_food)

Та же дизайн-система (шрифты Playfair Display + DM Sans, флип-карточки, модалка, pull-to-refresh, nav-вкладки, today-highlight), но **холодная «силовая» палитра**:

--card-back: #1f2a37

--accent-push:#e0a04e  (янтарный — PUSH)

## 7. База данных `history/training.db` (SQLite)

Аналог `menu_history.db`:

- **`training_weeks`** — `id, week_start, week_end, filename, push_level, pull_level, focus, summary, created_at`
- **`exercise_log`** — план vs факт по блокам (аналог `menu_dishes`):
  `id, week_id, date, window (утро/день/вечер), block (push/pull/mob), exercise, planned, done, felt (rpe/словом), note, liked`
- **`progress`** — динамика цифр (аналог `weight_history`):
  `id, recorded_at, metric, value, note`. Метрики: `max_pushup_wall`, `max_pushup_knee`, `max_hang_sec`, `max_active_hang_sec`, `max_row`, `max_negative_sec`, …
- **`training_rules`** — аналог `food_rules`:
  `id, category, name, rating, note, valid_until, source, created_at, updated_at`.
  Семантика `rating`: `-2` боль/жёсткий стоп (исключить или заменить), `-1` временно убрать (до `valid_until`), `+1` норм, `+2` «зашло — давать чаще».

## 8. Скилл `/make-week` (аналог `make-menu`)

Шаги:
1. Прочитать `profile.json` (текущие уровни, базовые цифры).
2. Прочитать `plan/progression.md` (лестницы + milestones + правила GtG).
3. Вычитать активные `training_rules` (без этого неделю не генерировать).
4. Прочитать последние `progress` и `exercise_log`.
5. Принять обратную связь пользователя за прошлую неделю (что удалось / что тяжело / что пропустил / где болело).
6. Решить: где +/− объём, где сменить уровень (по milestone), что заменить (по правилам `-2`/`-1`).
7. Сгенерировать `weeks/2026/week_YYYY-MM-DD.html` в стиле проекта (скопировать CSS/скрипты эталона, не изобретать дизайн).
8. Записать неделю и блоки в БД (`training_weeks`, `exercise_log`), обновить `progress` при новых рекордах.
9. Добавить карточку недели первой в `index.html`.
10. `git commit` + `git push`; вывести итог пользователю.

**Команды обратной связи** (как у `food_rules`):
| Пользователь говорит | Действие |
|---|---|
| «вис стал 30 сек» | `INSERT progress(metric='max_hang_sec', value=30)` |
| «плечо ноет на висе» | `training_rules` rating `-2` для виса + добавить мобилизацию плеча |
| «негативы зашли» | `training_rules` rating `+2` |
| «отжимания от стены легко» | пометка → скилл двинет на P1 |
| «во вторник пропустил день-блок» | `exercise_log.done=0/note` |

## 9. Экипировка (`equipment.md`)

**Турник:** есть, высокий, фиксированный. **Проверить нагрузку ≥150 кг** (при 130 кг + рывок нужен запас).

**Резинки — купить (петлевые/loop):**
| Лента | Ширина | Помощь | Назначение |
|---|---|---|---|
| Супер-тяжёлая (обязательно) | ~65 мм | ~30-70 кг | Подтягивания/негативы с большой помощью на старте |
| Экстра-тяжёлая (обязательно) | ~45 мм | ~23-54 кг | В связке с первой; одна — когда окрепнешь |
| Средняя (опц., позже) | ~22-32 мм | ~10-40 кг | Band rows, мобилити плеч, дальнейшая прогрессия |

Помощь ленты максимальна внизу (где слабее всего) — удобно. Резинки закрывают: band-assisted подтягивания, негативы, горизонтальные тяги, здоровье плеч.

**Опоры без покупок:**
- Отжимания P0-P2: стена → подоконник/стол → диван/устойчивый стул.
- Горизонтальные тяги: band rows (лента закреплена низко) или под крепким столом (проверить надёжность).
- Опционально позже: гимнастические кольца/петли на высокий турник — лямки опускаются на нужную высоту.

## 10. Деплой

Новый репозиторий + GitHub Pages, базовый путь `/push_pull-ups/` (как `/health_food/`). Все абсолютные пути в PWA-тегах и SW — с этим префиксом.

**Шаги пользователя (один раз):**
1. Создать пустой репозиторий `push_pull-ups` на GitHub.
2. `git remote add origin …` + `git push -u origin main`.
3. GitHub → Settings → Pages → Source: `main / (root)`.

`profile.json` — в `.gitignore` (как `profiles/` в health_food), не пушится.

## 11. Стартовое состояние (неделя 1 — калибровка)

Неделя 1 мягкая, задачи: поставить технику, нащупать максимумы (для расчёта рабочих 40-60%), завести привычку.
- PUSH: P0 от стены. Найти аккуратный максимум (без отказа), рабочие подходы ~50%.
- PULL: H0 вис + H1 активный вис. Замер: сколько держишь чисто.
- Мобилити: запястья + плечи ежедневно.
- Резинки, скорее всего, приедут к неделе 2 → H3-H5 подключаем, когда будут.

## 12. Открытые вопросы / на будущее
- Точные стартовые цифры рабочих подходов — калибруются в неделе 1 по факту.
- Кольца/петли на турник — отложенная опция.
- Возможный бейдж «серия дней подряд» (streak) для мотивации — не в первой версии (YAGNI).
